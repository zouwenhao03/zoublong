1.put和post请求有何不同

PUT和POST是HTTP协议中两种不同的请求方法，它们在使用上有一些关键的区别：

  1. 语义不同：

    • PUT： 用于向指定的URI（统一资源标识符）更新资源，或者在URI不存在的情况下创建一个新的资源。如果资源已经存在，PUT请求会对其进行更新；如果资源不存在，PUT请求可以创建一个新的资源。换句话说，PUT是幂等的，多次执行相同的PUT请求结果应该是相同的。
    • POST： 用于向指定URI提交数据，用来创建一个新的资源。与PUT不同，POST不具有幂等性，多次执行相同的POST请求可能会得到不同的结果。

  2. 安全性和幂等性：

    • PUT： 通常是幂等的，即多次执行相同的PUT请求应该具有相同的效果。但是，这并不是强制要求的，因为有些情况下，PUT可能用于生成新的资源标识，因此可能不是幂等的。
    • POST： 通常不是幂等的，因为多次执行相同的POST请求可能会创建多个相同的资源，尤其是在每次请求中包含了一些不同的数据时。

  3. 请求体：

    • PUT： 通常用于更新或创建资源，请求体中包含完整的资源表示。如果资源已经存在，它将被替换；如果资源不存在，将会创建一个新的资源。
    • POST： 通常用于提交数据，请求体中包含要处理的数据，服务器根据请求体中的数据创建新的资源。

  4. 使用场景：

    • PUT： 适用于更新已知资源，客户端提供完整的资源表示。例如，更新用户的个人信息。
    • POST： 适用于创建新资源，服务器生成资源标识，并根据请求体中的数据创建新资源。例如，提交表单数据以创建新的博客文章。

总体来说，PUT用于幂等的更新或创建资源，而POST用于非幂等的创建资源。选择使用哪种方法取决于你的具体需求和语义要求。

3.HTTP状态码及其含义

HTTP状态码是由服务器返回到客户端的3位数字代码，用于表示特定HTTP请求的结果。每个状态码都有特定的含义，分为五个不同的类别。以下是一些常见的HTTP状态码及其含义：

  1. 1xx系列（信息性状态码）：

    • 100 Continue： 服务器已经收到了请求的一部分，客户端可以继续发送剩余数据。
    • 101 Switching Protocols： 客户端请求服务器切换协议，例如切换到WebSocket。

  2. 2xx系列（成功状态码）：

    • 200 OK： 请求成功，服务器正常处理并返回请求的数据。
    • 201 Created： 请求已经被成功处理，并在服务器上创建了新的资源。
    • 204 No Content： 请求成功，但响应不包含实体的主体部分，通常用于删除操作。

  3. 3xx系列（重定向状态码）：

    • 301 Moved Permanently： 请求的资源已被永久移动到新的URI，客户端应该使用新的URI重新发起请求。
    • 302 Found： 请求的资源临时移动到新的URI，客户端应该继续使用原始的URI。
    • 303 See Other： 类似于302，但要求客户端使用GET请求获取资源。
    • 304 Not Modified： 资源未修改，可以使用缓存的版本。

  4. 4xx系列（客户端错误状态码）：

    • 400 Bad Request： 请求无效，服务器无法理解或处理请求的格式。
    • 401 Unauthorized： 请求要求身份验证，客户端没有提供有效的身份证书。
    • 403 Forbidden： 服务器理解请求，但拒绝执行，通常因为缺少权限。
    • 404 Not Found： 请求的资源不存在。

  5. 5xx系列（服务器错误状态码）：

    • 500 Internal Server Error： 服务器遇到了不可预知的错误，无法完成请求。
    • 501 Not Implemented： 服务器不支持请求的功能，或者不能完成请求。
    • 503 Service Unavailable： 服务器当前无法处理请求，通常是由于过载或维护导致的。

这些状态码帮助客户端了解其请求的结果，以便适当地采取后续的行动。

5.link和@import区别

  1. 区别：
    link属于html标签。@import在css中使用表示导入外部样式表；
    页面被加载的时，link会同时被加载，而@import引用的CSS会等到页面被加载完再加载;
    import只在IE5以上才能识别，而link是HTML标签，无兼容问题;
    link方式的样式的权重 高于@import的权重；
    link 支持使用javascript改变样式 （document.styleSheets），后者不可

  2. link和@impor的详细区别

    1. 从属关系
      1.1 link：link是HTML提供的标签，不仅可以加载CSS，还可以定义rel等属性
      1.2 @import：@import是css提供的语法，只有导入样式表的作用

  3. 加载顺序
    2.1 link：link在页面加载时CSS同时被加载
    2.2 @import：引入的CSS要等页面加载完毕后再加载

  4. 兼容性问题
    3.1 link是HTML提供的语法，不存在兼容性问题
    3.2 @import是css2.1提供的语法，ie5以上才兼容

  5. DOM可控性
    js控制DOM时，可以通过插入link标签来改变样式，不能通过@import改变

  6. 权重问题（有争议）
    link标签引入的样式权重大于@import标签

6.viewport

Viewport（视口）是指浏览器用来显示网页内容的区域，它可以是整个浏览器窗口或者是页面的一部分。在移动设备上，视口的概念变得尤为重要，因为移动设备的屏幕尺寸相对较小，需要适应不同设备的显示情况。

在HTML中，通过 <meta> 标签中的 viewport 属性可以对视口进行控制。常见的 viewport 属性包括：

  • width（宽度）： 设置视口的宽度，可以是具体的像素值，也可以是特殊值如 device-width 表示设备的屏幕宽度。

  • initial-scale（初始缩放比例）： 设置页面的初始缩放比例，通常设置为1.0。

  • minimum-scale（最小缩放比例）和maximum-scale（最大缩放比例）： 分别用于设置用户可以缩小或放大页面的程度。

  • user-scalable（用户是否可以缩放）： 控制用户是否可以手动缩放页面。如果将其设置为 “no”，则用户将无法手动缩放页面。

这些属性可以帮助开发者在移动设备上创建响应式的网页，使网页在不同设备上具有更好的可视和可操作性。合理设置视口可以确保网页在移动设备上以最佳方式呈现，适应不同屏幕尺寸和分辨率。
