2. 什么是可替换元素，什么是非可替换元素，它们各自有什么特点？

在 HTML 中，元素可以被分为可替换元素和非可替换元素两种类型。

可替换元素是指其内容完全由外部资源或者属性来决定，例如 <img>、<input>、<textarea>、<select> 等。可替换元素的特点包括：

  • 通常有固定的尺寸，例如图片的宽度和高度；
  • 可以通过 CSS 中的 vertical-align 属性来控制其垂直方向上的对齐方式；
  • 可以通过 HTML 中的 alt 属性来提供替代文本；
  • 可以被浏览器缓存，因为它们通常具有不变的内容；
  • 可以作为其他元素的替代品来使用。

非可替换元素是指其内容不受外部资源或属性影响，例如 <p>、<div>、<span>、<table> 等。非可替换元素的特点包括：

  • 通常没有固定的尺寸，它们的大小通常由其内容和 CSS 的样式来决定；
  • 可以作为其他元素的容器，承载其他元素；
  • 通常没有默认的行为和默认样式，需要通过 CSS 来控制其显示效果；
  • 可以被 JavaScript 用来操作 DOM。

3.src 和 href 的区别

在 HTML 中，src 和 href 属性都用于指定资源的路径，但它们有不同的含义和用法。

  • src（source）属性指向一个外部资源（通常是脚本、图像、视频等），用于将该资源嵌入到当前文档中。在嵌入资源时，浏览器会向指定的 URL 发送一个 GET 请求，并将响应的资源加载到当前文档中。在 JavaScript 中，可以通过 DOM 对象的 src 属性来获取或设置该元素的 src 属性值。

  • href（hypertext reference）属性指向一个超链接，用于定义一个文档之间或同一文档内的跳转链接。在点击一个包含 href 属性的元素时，浏览器会向指定的 URL 发送一个 GET 请求，然后将响应的文档加载到当前窗口或标签页中。在 JavaScript 中，可以通过 DOM 对象的 href 属性来获取或设置该元素的 href 属性值。

因此，简单地说，src 是用于加载一个嵌入的资源，而 href 是用于定义一个超链接。

4. 说说常用的 meta 标签

meta标签是HTML页面中的一种特殊标签，用于提供元数据，也可以用于告知浏览器如何展示页面。常用的meta标签包括：

  1. <meta charset="字符集">：指定文档使用的字符编码，通常设置为UTF-8。
  2. <meta name="viewport" content="width=device-width, initial-scale=1.0">：用于响应式布局，让网页自适应移动设备的屏幕大小，其中width=device-width指定了页面的宽度等于设备的宽度，initial-scale=1.0表示初始缩放比例为1。
  3. <meta name="keywords" content="关键词1,关键词2,...">：用于指定页面的关键词，有利于搜索引擎对页面内容的索引。
  4. <meta name="description" content="网页描述">：用于指定页面的描述信息，有利于搜索引擎对页面内容的索引。
  5. <meta name="robots" content="index,follow">：用于控制搜索引擎对页面的索引和跟踪，常用的取值包括index、noindex、follow和nofollow。
  6. <meta http-equiv="refresh" content="秒数;url=跳转地址">：用于定时刷新页面或者跳转到指定的页面地址。
  7. <meta http-equiv="X-UA-Compatible" content="IE=edge,chrome=1">：用于指定浏览器的兼容模式，通常设置为IE=edge,chrome=1，表示优先使用Chrome的渲染方式。
  8. <meta name="format-detection" content="telephone=no">：用于禁止iOS设备自动识别电话号码，避免电话号码被误认为是电话链接。
  9. <meta name="apple-mobile-web-app-capable" content="yes">：用于将网站添加到iOS设备的主屏幕，使得网站可以像原生应用一样在全屏模式下运行。
  10. <meta name="apple-touch-icon" href="图标路径">：用于指定添加到iOS设备主屏幕的图标，通常设置为一个144x144像素的png格式图片。

5.对 html 语义化的理解

HTML 语义化是指根据 HTML 标签的语义或含义来编写 HTML 代码，使得代码结构更加清晰，易于理解和维护，同时也有利于搜索引擎的优化。它可以使得网页的结构更加清晰明了，增加代码的可读性和可维护性，同时也有利于网页的SEO优化。

语义化的 HTML 代码应该符合以下几个方面：

  1. 使用正确的 HTML 标签来标记不同的内容，如使用 <nav> 标签来标记导航栏，使用 <article> 标签来标记文章内容等。

  2. 标题应该按照递增的顺序（从 H1 到 H6）来标记，使得标题的层次结构清晰明了。

  3. 使用语义化的命名方式来为 HTML 标签添加 class 和 id 属性，如将导航栏的 class 设置为 nav。

  4. 避免使用纯样式标签（如 <font>、<b>、<i> 等），而是使用 CSS 来实现样式的控制。

  5. 为图像等元素添加 alt 属性，以方便搜索引擎识别和无障碍访问。

总之，HTML 语义化是一种良好的编程习惯和规范，能够让代码更易于阅读和维护，同时也有利于网站的可访问性和搜索引擎优化。

6. HTML 与 XHTML 二者有什么区别，你觉得应该使用哪一个并说出理由

HTML（HyperText Markup Language）和XHTML（Extensible HyperText Markup Language）都是用于创建Web页面的标记语言。它们的主要区别在于语法规则和解析方式。

HTML是一种基于SGML（标准通用标记语言）的标记语言，它的语法相对宽松，允许一些不规范的写法，比如标签可以不关闭、属性值可以不使用引号等。HTML5标准已经逐渐摆脱了这些不规范的写法，变得更加严谨。

XHTML则是一种基于XML（可扩展标记语言）的标记语言，它的语法更加严格，标签必须被正确地嵌套，属性必须使用引号括起来，标签必须使用小写字母等。XHTML的语法规则使得它更容易被计算机处理，更符合XML的规范。

对于使用哪一个标记语言，这取决于具体情况。在早期，XHTML因为更加严格的语法规则，对于一些需要对页面进行严格控制的场景（如搜索引擎优化）更为适用。但是现在HTML的语法规范已经不断发展完善，而且现代浏览器对HTML的支持也越来越好，所以在一般的Web开发中，使用HTML已经足够了。而对于一些特殊场景，如需要向老旧浏览器提供服务的情况，可能需要使用XHTML。

7.行内元素和块级元素区别，有哪些，怎样转换？

HTML中的元素可以被归为两类：行内元素和块级元素。它们有以下区别：

  1. 块级元素默认情况下会占据一整行，而行内元素只占据它所需的宽度和高度。
  2. 块级元素可以包含行内元素和其他块级元素，而行内元素不能包含块级元素，只能包含其他行内元素和文本。
  3. 块级元素可以设置宽度、高度、内边距和外边距等样式属性，而行内元素只能设置部分样式属性，无法设置宽高，只能设置左右内外边距，如字体颜色和背景颜色等。
  4. 块级元素会自动换行，而行内元素不会。

有些元素既可以是块级元素，也可以是行内元素，这些元素被称为“行内块元素”，例如<img>和<input>。

在实际开发中，根据具体的需求和场景选择使用块级元素还是行内元素或行内块元素。如果需要让元素占据整行或设置元素的大小、内边距和外边距等，应该使用块级元素。如果元素只包含文本或其他行内元素，应该使用行内元素。有时候需要将行内元素转换成块级元素或者将块级元素转换成行内元素，可以使用CSS中的display属性来实现，例如将行内元素<a>转换成块级元素可以使用display:block，将块级元素<div>转换成行内元素可以使用display:inline。

8.介绍下 BFC 及其应用

BFC（Block Formatting Context）指的是块级格式化上下文，是 Web 页面中的一种 CSS 渲染模式。BFC 规定了内部块级元素的布局规则，对浮动元素、清除浮动等布局问题提供了有效的解决方案。

BFC 的特点：

  1. 内部的 Box 会在垂直方向，一个接一个地放置。
  2. 在一个 BFC 中，两个相邻 Box 的垂直间距是由它们的 margin 决定的，属于 margin collapsing 的情况不会出现。
  3. BFC 的区域不会与浮动元素的 box 重叠。
  4. BFC 在页面上是一个独立的容器，外部元素的 margin 不会影响到它内部的元素，反之亦然。
  5. 计算 BFC 的高度时，浮动元素也会参与计算。

BFC 的应用场景：

  1. 解决 margin collapsing 问题
  2. 清除浮动，避免父元素高度塌陷
  3. 防止元素间的重叠
  4. 实现多栏布局
  5. 实现自适应布局

要创建 BFC，可以使用以下方式之一：

  1. float 属性不为 none
  2. position 属性为 absolute 或 fixed
  3. display 属性为 inline-block、table-cell、table-caption、flex、inline-flex 或 grid
  4. overflow 属性不为 visible

9.分析比较 opacity: 0、visibility: hidden、display: none 优劣和适用场景

这三个 CSS 属性分别用于隐藏元素，它们的作用机制不同，因此适用的场景也有所不同：

  1. opacity: 0 是非继承属性

这个属性设置元素的透明度为 0，使得元素完全透明，但仍然占据着页面上的空间。这意味着该元素隐藏了，但是仍然可以对它进行交互（如点击）。因此，该属性适用于需要占据空间但不需要展示的元素。

  1. visibility: hidden 是继承属性

该属性将元素设为不可见，但是它仍然会占据着页面上的空间，并且会影响布局。这意味着该元素隐藏了，但仍然保留了自己的位置和大小。因此，适用于需要隐藏元素但保留位置和大小的场景。不会让元素从渲染树消失，渲染元素继续占据空间，只是内容不可见，不能点击

  1. display: none 是非继承属性

该属性将元素从页面中完全删除，即该元素在页面上不再存在，不会影响布局，不占据任何空间，也无法对其进行交互。因此，适用于需要完全隐藏元素，且不影响页面布局和交互的场景。

综上所述，三种属性适用的场景不同，具体应该根据实际需求进行选择。

11.居中为什么要使用 transform（为什么不使用 marginLeft/Top）

使用 transform 居中相对于使用 margin-left 和 margin-top 来居中的优势在于：

  1. 灵活性：transform可以对元素进行旋转、缩放等变换，而margin只能控制元素的位置，使用transform更加灵活。
  2. 性能：使用transform可以获得更好的性能，因为transform可以利用 GPU 加速，而margin则只能由 CPU 处理。
  3. 可靠性：使用transform可以获得更加可靠的居中效果。使用margin进行居中的话，需要知道元素的尺寸，而如果元素的尺寸发生变化，就需要重新计算margin的值，而使用transform则可以不用考虑元素的尺寸，始终能够获得居中效果。
  1. 兼容性：使用transform可以获得更好的浏览器兼容性，因为transform是 CSS3 新增的特性，而margin在一些比较老的浏览器中可能会有兼容性问题。

综上所述，使用transform进行元素居中是一种更加灵活、性能更好、可靠性更高、兼容性更好的方案。

注释：通常，使用 transform 属性对元素进行移动、旋转和缩放等变换时，浏览器会使用 GPU 来执行这些操作，因为 GPU 专门用于处理图形和图像，而且拥有比 CPU 更强大的计算能力。

相比之下，使用 marginLeft 和 marginTop 等属性对元素进行居中定位时，浏览器会使用 CPU 来计算元素的位置，这可能会导致性能下降，特别是在处理大量元素时。因此，使用 transform 属性可以提高性能并改善用户体验。

GPU是英文Graphics Processing Unit的缩写，即图形处理器。它是计算机系统中的一种特殊处理器，专门用于处理图形和图像的计算，通常与中央处理器（CPU）协同工作，加速计算机处理图形、图像和视频等方面的能力。GPU比CPU更适合进行并行计算，因此被广泛应用于游戏、图像处理、科学计算等领域。

综上所述，使用 transform 属性进行居中可以获得更好的效果和性能。

13.什么是视口？

viewport 即视窗、视口，用于显示网页部分的区域，在 PC 端视口即是浏览器窗口区域，在移动端，为了让页面展示更多的内容，视窗的宽度默认不为设备的宽度，在移动端视窗有三个概念：布局视窗、视觉视窗、理想视窗

  • 布局视窗：在浏览器窗口 css 的布局区域，布局视口的宽度限制 css 布局的宽。为了能在移动设备上正常显示那些为 pc 端浏览器设计的网站，移动设备上的浏览器都会把自己默认的 viewport 设为 980px 或其他值，一般都比移动端浏览器可视区域大很多，所以就会出现浏览器出现横向滚动条的情况

  • 视觉视窗：终端设备显示网页的区域

  • 理想视窗：针对当前设备最理想的展示页面的视窗，不会出现横向滚动条，页面刚好全部展现在视窗内，理想视窗也就是终端屏幕的宽度。
