21.清除浮动的方法

清除浮动是为了解决浮动元素引起的父元素高度塌陷或宽度不正常，常用的清除浮动的方法有以下几种：

  1. 给浮动元素父级设置高度
  2. 父级同时浮动（需要给父级同级元素添加浮动）
  3. 父级设置成 inline-block，其 margin: 0 auto 居中方式失效
  4. 给父级添加 overflow:hidden 清除浮动方法
  5. 万能清除法 after 伪类清浮动（现在主流方法，推荐使用）

22.伪元素和伪类的区别和作用？

伪元素和伪类是 CSS 中用来对文档的一些特定部分进行样式设置的两种不同的机制。

伪元素（pseudo-elements）用于创建一些不在文档树中的元素，并为其添加样式。它们以双冒号 :: 开头，常见的有 ::before 和 ::after 等。伪元素在元素的内容区域之前或之后创建了一个虚拟的元素，并且可以通过 CSS 来设置它的样式、内容和位置等。例如：

23.重绘与重排的区别？

重绘（Repaint）和重排（Reflow）是浏览器渲染页面时的两个重要概念。

重绘指的是当元素样式的改变不影响它在文档流中的位置和大小时，浏览器会根据新样式重新绘制该元素的内容，但不会改变元素的布局信息和文档流结构。例如，改变一个元素的背景色或字体颜色，都只会触发重绘，而不会改变元素的大小和位置。

重排指的是当页面布局和元素大小发生改变时，浏览器需要重新计算元素的位置和大小，以及对整个文档流的布局信息进行重新排版。例如，改变一个元素的宽度、高度、边距、位置等属性都会触发重排，因为这些改变会影响其他元素的位置和文档流结构。

由于重排需要重新计算元素的大小和位置，因此比重绘的成本更高。在实际开发中，应该尽量减少重排和重绘的次数，以提高页面性能。一些常见的优化技巧包括：

  • 避免重排或重绘的方法包括：

    1. 避免频繁读取和修改样式；
    2. 将要修改的元素脱离文档流；
    3. 使用虚拟文档片段；
    4. 批量修改DOM；
    5. 使用transform；

    以上这些方法的目的都是尽量减少对DOM的操作，从而避免触发重排或重绘。同时，还可以使用其他的性能优化方法，例如减少HTTP请求次数、压缩文件大小等等。

24.优化图片的方法

以下是优化图片的一些方法：

  1. 对于很多装饰类图片，尽量不用图片，因为这类修饰图片完全可以用 CSS 去代替。
  2. 对于移动端来说，屏幕宽度就那么点，完全没有必要去加载原图浪费带宽。一般图片都用 CDN 加载，可以计算出适配屏幕的宽度，然后去请求相应裁剪好的图片。
  3. 小图使用 base64 格式
  4. 雪碧图
  5. 选择正确的图片格式：
  • 对于能够显示 WebP 格式的浏览器尽量使用 WebP 格式。因为 WebP 格式具有更好的图像数据压缩算法，能带来更小的图片体积，而且拥有肉眼识别无差异的图像质量，缺点就是兼容性并不好
  • 小图使用 PNG，其实对于大部分图标这类图片，完全可以使用 SVG 代替
  • 照片使用 JPEG

25.style 标签写在 body 后面与 body 前面有什么区别？

页面加载自上而下，当然是先加载样式。写在 body 标签后由于浏览器以逐行方式对 HTML 文档进行解析，当解析到写在尾部的样式表（外联或写在 style 标签）会导致浏览器停止之前的渲染，等待加载且解析样式表完成之后重新渲染

27.盒模型

CSS盒模型是指用于描述HTML元素在页面上所占空间的一种模型。CSS盒模型将一个HTML元素看作一个矩形盒子，这个盒子由四个部分组成：内容区域、内边距区域、边框区域和外边距区域。

其中，内容区域指的是元素的实际内容，如文本、图片等；内边距区域是内容区域与边框之间的空间，可以用padding属性进行设置；边框区域则是围绕着内容区域和内边距区域的线框，可以用border属性进行设置；外边距区域则是边框与相邻元素之间的空间，可以用margin属性进行设置。

在标准盒模型中，元素的宽度和高度仅包括内容区域的宽度和高度；而在IE盒模型中，元素的宽度和高度包括了内容区域、内边距和边框的宽度和高度。可以通过CSS的box-sizing属性来设置元素使用哪种盒模型。

28.如何实现垂直居中？有哪些方法？

实现垂直居中有多种方法，以下是其中的几种：

  1. 使用 flexbox：使用 display: flex 和 align-items: center 可以将子元素垂直居中。
  1. 使用绝对定位和 transform：使用 position: absolute 和 transform 属性可以将子元素垂直居中。需要将父元素设置为相对定位。
  1. 使用 table 和 table-cell：将父元素设置为 display: table，将子元素设置为 display: table-cell，再使用 vertical-align: middle 可以将子元素垂直居中。
  1. 使用 grid：使用 display: grid 和 align-items: center 可以将子元素垂直居中。

注意：以上的方法并不是适用于所有情况的，需要根据具体场景进行选择和调整。

30.如何实现无限滚动

无限滚动指的是当用户滚动到页面底部时，自动加载更多内容，以此实现页面的无限滚动。常见的实现无限滚动的方法包括以下几种：

  1. 监听滚动事件，在滚动到底部时发送异步请求，获取更多数据，并将数据插入到页面中。这种方法的缺点是每次滚动到底部都会触发一次请求，可能会导致页面卡顿。

  2. 使用 Intersection Observer API，该 API 可以监听元素与视窗的交叉状态，可以实现在元素进入视窗时触发回调函数的功能。使用该 API 可以实现在用户滚动到页面底部时触发加载更多数据的功能。该方法的优点是性能更好，可以减少不必要的请求。

  3. 使用第三方库，如 jQuery、Infinite Scroll 等，这些库封装了无限滚动的实现细节，可以方便地实现无限滚动功能。但是使用第三方库需要考虑文件大小和兼容性等问题。

总之，实现无限滚动需要注意性能问题，避免不必要的请求，保证用户体验。

31.iframe的作用以及优缺点

iframe是一种HTML标签，用于在一个网页中嵌入另一个网页。它的作用主要有两个：

  1. 实现网页嵌套：可以在一个网页中嵌入另一个网页，方便显示不同来源的内容。
  2. 实现异步加载：可以通过iframe异步加载其他网页的内容，避免影响当前页面的加载和渲染。

iframe的优点：

  1. 网页嵌套：通过iframe可以在一个网页中嵌套另一个网页，方便显示不同来源的内容。
  2. 异步加载：通过iframe异步加载其他网页的内容，可以避免阻塞当前页面的加载和渲染。

iframe的缺点：

  1. SEO问题：由于搜索引擎不会对iframe内部的内容进行解析，所以iframe内部的内容不利于搜索引擎优化。
  2. 安全问题：iframe可以跨域访问，如果不加以限制，可能会被用于恶意攻击。
  3. 可维护性问题：由于iframe中的内容是从外部加载的，如果外部网页发生变化，可能会影响到iframe中的内容，增加维护难度。

因此，在使用iframe时需要注意安全和SEO问题，并考虑是否真正需要使用它。
