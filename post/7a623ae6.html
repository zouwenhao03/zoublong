1.JavaScript 有哪些数据类型，它们的区别？

JavaScript 有以下数据类型：

  1. 基本数据类型（Primitive Types）：

    • Number：表示数字，包括整数和浮点数，如：3, 3.14。
    • String：表示字符串，如：”hello”。
    • Boolean：表示逻辑值，true 或 false。
    • Null：表示空值，仅有一个值 null。
    • Undefined：表示未定义的值，仅有一个值 undefined。
    • Symbol：表示唯一的标识符，用于对象属性名等。
    • BigInt:是 ECMAScript 2020 中引入的一种新的基本数据类型，用于表示任意精度的整数。

  2. 引用数据类型（Reference Types）：

    • Object：表示对象，是一种复合数据类型，可用于存储多个键值对，如：{name: “Alice”, age: 18}。
    • Array：表示数组，是一种特殊的对象，用于存储有序的数据集合，如：[1, 2, 3]。
    • Function：表示函数，是一种可执行的对象，如：function add(a, b) {return a + b;}。
    • Date：表示日期，如：new Date()。
    • RegExp：表示正则表达式，如：/abc/。

基本数据类型与引用数据类型的区别在于它们的存储方式不同。基本数据类型的值是直接存储在变量访问的位置上，而引用数据类型的值是存储在堆内存中的对象，变量实际上是一个指向该对象的指针。因此，基本数据类型的赋值是值传递，而引用数据类型的赋值是引用传递。

1.JavaScript 中如何判断一个值的数据类型？

JavaScript 中可以使用 typeof 运算符来判断一个值的数据类型，它返回一个字符串，表示该值的数据类型。例如，typeof 42 返回 “number”，typeof “hello” 返回 “string”，typeof true 返回 “boolean”。需要注意的是，typeof null 返回 “object”，这是一个历史遗留问题，而不是一个 Bug。

2.什么是类型转换？在 JavaScript 中如何进行类型转换？

类型转换是指将一个数据类型的值转换为另一个数据类型的值。在 JavaScript 中，有两种类型转换：隐式类型转换和显式类型转换。隐式类型转换是指 JavaScript 引擎自动将数据类型进行转换，如在字符串中使用 + 运算符连接一个数字时，会将数字隐式转换为字符串。显式类型转换是指通过代码明确地将一个数据类型转换为另一个数据类型，如使用 Number() 函数将字符串转换为数字。常见的类型转换函数包括 Number()、String()、Boolean()、parseInt()、parseFloat() 等。

关于隐式转换

在 JavaScript 中，隐式类型转换是由 JavaScript 引擎自动完成的。隐式类型转换遵循一些规则，以下是常见的隐式类型转换规则：

  1. 字符串拼接：如果其中一个操作数是字符串类型，另一个操作数会隐式转换为字符串类型。

  2. 数学运算：如果操作数都是数字类型，则执行数学运算；如果其中一个操作数不是数字类型，则将其隐式转换为数字类型。

  3. 比较运算：如果两个操作数类型不同，则将其中一个操作数隐式转换为另一个操作数的类型，然后再执行比较运算。

  4. 逻辑运算：如果操作数是非布尔类型，则将其隐式转换为布尔类型。

  5. 类型转换函数：使用 Number()、String()、Boolean() 等类型转换函数将值显式地转换为特定的类型。

需要注意的是，隐式类型转换的优先级可能会导致一些意外的结果，因此在进行类型转换时，需要注意数据类型的精度问题和运算符的优先级等因素。为了避免隐式类型转换的问题，可以使用严格相等运算符（===）来比较值的类型和值是否完全相等。

3.数据类型检测的方式有哪些

在 JavaScript 中，常用的数据类型检测方式有以下几种：

  1. typeof 运算符：用于检测变量或值的数据类型，返回一个字符串，表示变量或值的数据类型。例如：typeof "hello" 返回 "string"，typeof 123 返回 "number"，typeof true 返回 "boolean"，typeof [] 返回 "object"，typeof null 返回 "object"，typeof undefined 返回 "undefined"。

    但它无法区分 null 和对象，返回值均为 "object"。

  2. instanceof 运算符：用于检测对象是否是某个类的实例，返回一个布尔值。例如：[] instanceof Array 返回 true，{} instanceof Object 返回 true，"hello" instanceof String 返回 false。

    无法检测基本类型和对象字面量等其他数据类型。

  3. Object.prototype.toString.call() 方法：用于返回变量或值的数据类型，返回一个字符串，表示变量或值的数据类型。与 typeof 运算符相比，可以更准确地检测对象的数据类型。例如：Object.prototype.toString.call("hello") 返回 "[object String]"，Object.prototype.toString.call([]) 返回 "[object Array]"。

    语法比较繁琐

  4. Array.isArray() 方法：用于检测一个变量或值是否为数组，返回一个布尔值。例如：Array.isArray([]) 返回 true，Array.isArray({}) 返回 false。

    只能检测一个变量或值是否为数组，无法检测其他数据类型

需要注意的是，不同的数据类型检测方式适用于不同的场景，需要根据具体情况选择合适的方式。

4.变量声明

在 JavaScript 中，变量可以使用 var、let 或 const 进行声明。

  • 在ES6之前，var 是 JavaScript 中声明变量的主要方式。var 的特点如下：

    1. 变量提升：在同一个作用域内，无论你在哪里声明变量，它都会被提升到该作用域的顶部，因此可以在声明之前使用变量。

    2. 全局作用域：var 声明的变量在全局作用域中，因此容易造成命名冲突，而且很难追踪变量的来源。

    3. 函数作用域：在函数内部使用 var 声明变量时，该变量只在该函数内部可见，外部无法访问。这种作用域也被称为函数作用域。

    4. 可重复声明：可以在同一作用域内重复声明同一个变量，后面的声明会覆盖前面的声明。

    5. 没有块级作用域：var 声明的变量在任何位置都可以被访问到，即使是在循环、条件语句等块级作用域内部也可以访问到，因此可能会出现变量污染的问题。

  • let 是 ES6 中引入的新的变量声明关键字，与 var 不同，它具有以下特点：

    1. 块级作用域：使用 let 声明的变量具有块级作用域，它们只在当前代码块内有效。在块级作用域外无法访问该变量。而使用 var 声明的变量则是函数作用域或全局作用域的变量，可能会导致变量污染。

    2. 不允许重复声明：在同一个作用域内，使用 let 声明的变量不能被重复声明，否则会报错。而使用 var 声明的变量可以被重复声明，后面的声明会覆盖前面的声明。

    3. 暂时性死区：使用 let 声明的变量会形成暂时性死区（Temporal Dead Zone，简称 TDZ），在该变量声明之前，任何访问该变量的操作都会报错。这种特性可以避免变量提升导致的意外行为，也可以使代码更加规范、易于维护。

    4. 不会被提升：使用 let 声明的变量不会被提升到代码块的顶部，因此在声明之前访问该变量会报错。而使用 var 声明的变量会被提升到函数作用域或全局作用域的顶部，因此可以在声明之前访问该变量。

    5. 支持循环作用域：在 for 循环中使用 let 声明的变量具有循环作用域，每次迭代都会创建一个新的变量，因此不会出现变量污染的问题。

  • const 是 ES6 中引入的一个新的声明变量的方式，它具有以下特点：

    1. 块级作用域：const 声明的变量和 let 声明的变量一样，只在声明的块级作用域内有效。

    2. 不可重复声明：在同一作用域内，不允许重复声明同一个变量，否则会报错。

4.箭头函数

JavaScript 箭头函数是 ES6 引入的一种新的函数定义方式，它有以下几个特点：

  1. 箭头函数使用箭头符号 => 来定义函数，语法比传统的函数定义更加简洁。
  2. 箭头函数是匿名函数，不能使用函数名来调用。
  3. 箭头函数**没有自己的 this、arguments、super 和 new.target**，它们会从定义它们的外层作用域继承这些值。
  4. 箭头函数的 this 始终指向词法作用域中的 this，而不是在运行时动态绑定 this，因此在使用 this 时需要格外注意。
  5. 箭头函数不能用作构造函数，不能使用 new 关键字实例化。
  6. 箭头函数没有 prototype 属性，因此无法定义原型方法。
  7. 箭头函数不能使用 yield 关键字，因此不能用作生成器函数。

总的来说，箭头函数具有代码简洁、作用域继承、不绑定 this 等特点，适用于许多函数式编程的场景。但也需要注意它们的局限性和使用注意事项。

7.isNaN 和 Number.isNaN 函数的区别？

isNaN 和 Number.isNaN 都是用于检查是否为 NaN 的函数，它们的区别在于：

  • isNaN 函数在检查之前会尝试将参数转换为数字，如果参数不能转换为数字，则返回 true。这意味着，如果参数是一个非数字的字符串或对象，或者是一个空字符串，**isNaN 函数都会返回 true**，这可能会导致误判。
  • Number.isNaN 函数不会尝试将参数转换为数字，而是只对 NaN 值返回 true，对于其他值，包括非数字的值和空值，都返回 false。

8.|| 和 && 操作符的返回值？

JavaScript 中的 || 和 && 操作符在逻辑表达式中使用。它们的返回值是一个布尔值或者操作符的操作数之一。

当使用 || 操作符时，如果左操作数为真，则返回左操作数，否则返回右操作数。如果两个操作数都是假值，则返回右操作数。

当使用 && 操作符时，如果左操作数为假，则返回左操作数，否则返回右操作数。如果两个操作数都是真值，则返回右操作数。

需要注意的是，当操作数不是布尔类型时，它们会被强制转换为布尔值。对于 || 操作符，任何可以转换为 true 的值都被视为真值，包括非零数值、非空字符串和对象。对于 && 操作符，任何可以转换为 false 的值都被视为假值，包括零值、空字符串、null 和 undefined。

11.map和Object的区别

Map 和 Object 都可以用来存储数据，但它们有一些区别：

  1. 键的类型： Map 的键可以是任意数据类型（包括基本类型和引用类型），而 Object 的键只能是字符串或者 Symbol 类型。

  2. 键值对个数： Map 的键值对个数可以通过 size 属性获取，而 Object 的键值对个数无法直接获取，需要通过遍历属性或者计算 Object.keys() 的长度来获取。

  3. 迭代顺序： Map 的迭代顺序是按照插入顺序依次迭代，而 Object 的属性是无序的，遍历时可能会随机打乱顺序。

  4. 性能： 在数据量较大时，Map 的性能优于 Object，因为 Map 内部使用了红黑树等数据结构，而 Object 的内部实现是哈希表，当数据量较大时，哈希表的性能会有所下降。

  5. API： Map 提供了许多方便的 API，如 set、get、has、delete、clear、keys、values、entries 等方法，而 Object 的 API 相对较少，只有 Object.keys()、Object.values()、Object.entries()、Object.assign() 等方法。

因此，在需要存储任意数据类型、需要按照插入顺序迭代、需要较高的性能、需要方便的 API 操作时，可以选择使用 Map；在只需要存储字符串或 Symbol 类型的键、不需要按照插入顺序迭代、数据量较小或者不需要高性能、需要简单的 API 操作时，可以选择使用 Object。

12.map和weakMap的区别

Map 和 WeakMap 的区别如下：

  1. 引用键值：Map 对象的键可以是任何值（包括基本类型和引用类型），而 WeakMap 对象的键必须是对象引用（也就是只能是对象）。

  2. 内存管理：在 Map 中，当引用类型的键不再使用时，仍会占用内存，因为它们仍然存在于 Map 中。而在 WeakMap 中，当引用类型的键不再被其他对象引用时，将自动从 WeakMap 中删除，从而释放内存。

  3. 迭代器：Map 具有可迭代性，而 WeakMap 没有，因为其键是随时可能被删除的。

  4. 方法：Map 对象有许多实用方法，如 size 属性、has()、get()、set() 和 delete() 等，而 WeakMap 只有 has()、get()、set() 和 delete() 四个方法。

总的来说，Map 适用于需要在键和值之间建立一对一关系，并且需要快速访问这些值的情况，而 WeakMap 适用于需要跟踪对象而不影响垃圾回收机制的情况。

13.js类数组对象

在 JavaScript 中，类数组对象是一种类似数组的对象，它们有一个 length 属性和一组从0开始的数字索引属性，但它们不具备数组对象的所有方法，例如 forEach()、map()、reduce() 等方法。

常见的类数组对象包括函数的参数对象 arguments，DOM 节点列表 NodeList 和 HTMLCollection，以及通过 document.querySelectorAll() 获取的 NodeList 集合等。

类数组对象并不是数组对象，它们没有数组的方法和属性，所以不能直接调用数组方法。

将类数组对象转成数组有以下几种方法：

  1. **Array.from()**：使用Array.from()方法将类数组对象转成数组。
  1. Array.prototype.slice.call()：使用Array.prototype.slice.call()方法将类数组对象转成数组。
  1. 扩展运算符：使用扩展运算符(…)将类数组对象转成数组。
  1. for…of循环：使用for…of循环将类数组对象转成数组。

需要注意的是，类数组对象的转换并不是万能的，对于某些特殊的类数组对象，可能需要使用其他的方法来转换成数组。

14.DOM和BOM

DOM（Document Object Model）和BOM（Browser Object Model）都是浏览器提供的API。

DOM是一种用于访问HTML和XML文档的编程接口，它将整个页面抽象为一个文档树，通过DOM提供的API可以对树上的节点进行操作，如增删改查等。

BOM则是浏览器提供的与浏览器窗口、浏览器操作相关的对象和方法的集合。BOM提供了与浏览器窗口、浏览器导航等相关的API，如window对象、location对象、history对象、navigator对象等。

15.escape、encodeURI、encodeURIComponent 的区别

escape、encodeURI 和 encodeURIComponent 都是 JavaScript 中用于编码 URL 的方法。

它们的区别在于：

  1. escape 方法用于编码字符串中的字符，它不会编码 ASCII 字母、数字和某些符号，包括 @ * _ + - . /。这些符号在 URL 中是可以使用的，但是 escape 不会对它们进行编码。escape 会将非 ASCII 字符转义为十六进制转义序列，其中每个字节使用 %XX 表示，XX 是两个十六进制数。但是 escape 方法已经被废弃，不推荐使用。
  2. encodeURI 方法用于编码整个 URL，但不会编码 URL 中的特殊字符，例如 :/?#[]@!$&'()*+,;=。而 encodeURI 会编码其他一些字符，例如 ö 会被编码为 %C3%B6。这些字符被认为是在 URL 中有特殊含义的字符，如果不进行编码，会导致 URL 解析错误。
  3. encodeURIComponent 方法用于编码 URL 中的参数值，所有非字母数字字符都会被编码。例如，encodeURIComponent('http://example.com/?param=value') 将返回 http%3A%2F%2Fexample.com%2F%3Fparam%3Dvalue。

因此，一般情况下，如果要编码整个 URL，应该使用 encodeURI 方法，如果要编码 URL 中的参数值，应该使用 encodeURIComponent 方法。不推荐使用 escape 方法。
