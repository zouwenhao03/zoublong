16.对AJAX的理解，实现一个AJAX请求

AJAX（Asynchronous JavaScript and XML）指的是一种通过 JavaScript 和 XML 进行异步通信的技术，它可以在不刷新整个页面的情况下向服务器发送请求，并获取到服务器返回的数据。它的优点是能够提高页面的用户体验，避免了整个页面的重新加载，只刷新部分页面。

在 JavaScript 中实现 AJAX 可以通过 XMLHttpRequest 对象来完成。下面是一个简单的 AJAX 请求示例：

首先创建一个 XMLHttpRequest 对象，然后监听其 onreadystatechange 事件，当 readyState 值为 4（即请求完成）并且 status 值为 200 时，表示请求成功，此时可以获取服务器返回的数据。

通过 open() 方法设置请求的类型（GET 或 POST）、URL 和异步标志（true 表示异步，false 表示同步），然后调用 send() 方法发送请求。在请求过程中可以通过 setRequestHeader() 方法设置请求头。

除了使用原生的 XMLHttpRequest 对象，还可以使用基于 Promise 的 fetch API、jQuery 中的 AJAX 方法等来实现 AJAX 请求。

17.ES6模块与CommonJS模块有什么异同？

ES6模块与CommonJS模块有以下异同点：

异同点：

  1. 导入和导出方式不同：ES6模块使用 import 和 export 语句进行导入和导出，而CommonJS模块使用 require() 导入，module.exports 或 exports 导出；
  2. 加载方式不同：ES6模块是静态的，编译时就能确定模块的依赖关系，使用 import 加载模块。而CommonJS模块是动态的，运行时才能确定依赖关系，使用 require() 加载模块；
  3. ES6模块导入的变量是只读的，无法修改变量的值，而CommonJS模块导入的变量是可以修改的；
  4. CommonJS模块是单个值导出，ES6模块可以导出多个变量或对象。

ES6模块的优点：

  1. 静态加载，在编译时就能确定模块的依赖关系，有助于编译器进行优化；
  2. 支持循环引用；
  3. 模块默认采用严格模式。

CommonJS模块的优点：

  1. 动态加载，支持在运行时动态加载模块，更适合在服务器端使用；
  2. 代码简单，语法简单，易于使用。

18.use strict是什么意思 ? 使用它区别是什么？

“use strict” 是 JavaScript 的严格模式，它是 ECMAScript 5 引入的一项新特性。使用它的目的是让 JavaScript 引擎在更严格的条件下运行代码，从而避免出现一些潜在的问题，提高代码的安全性和效率。

“use strict” 的使用有以下区别：

  1. 严格模式下禁止使用未声明的变量，否则会抛出错误。

  2. 严格模式下禁止删除变量、函数或参数。

  3. 严格模式下禁止变量重名，否则会抛出错误。

  4. 严格模式下禁止使用八进制字面量，否则会抛出错误。

  5. 严格模式下禁止使用 eval() 函数创建变量或函数，否则会抛出错误。

  6. 严格模式下禁止使用 with 语句，否则会抛出错误。

  7. 严格模式下对函数的 arguments 对象的访问更加严格，它不能被重新赋值，也不能用 delete 删除其中的某个参数。

  8. 严格模式下 this 的值为 undefined，而不是全局对象。

在 ES6 模块中，严格模式是默认启用的，所以无需在每个模块中添加 “use strict”。而在 CommonJS 模块中，需要手动添加 “use strict” 来启用严格模式。

20.js中的this

在 JavaScript 中，this 关键字通常用来引用当前函数运行时的上下文对象。this 的指向取决于函数的调用方式和位置，不同情况下可能会有不同的值。

  1. 全局上下文中的 this：在全局上下文中，this 指向全局对象（浏览器中为 window 对象，Node.js 中为 global 对象）。

  2. 函数上下文中的 this：在函数内部，this 的值取决于函数被调用时的上下文。通常情况下，this 指向调用函数的对象。如果函数被作为全局函数调用，则 this 指向全局对象。如果函数作为对象的方法调用，则 this 指向该对象。如果函数使用 call() 或 apply() 显式地指定了 this 的值，则 this 将指向所传递的对象。

  3. 构造函数中的 this：在构造函数中，this 指向当前正在创建的对象实例。

  4. 箭头函数中的 this：在箭头函数中，this 的值取决于函数被定义时的上下文，而不是调用时的上下文。箭头函数的 this 始终指向其外部作用域的 this 值。

总之，this 的值很重要，可以影响函数的行为和结果。因此，理解 this 的行为是很重要的。

22.面向对象

1.对象创建的方式

JavaScript 中有多种创建对象的方式，常见的包括以下几种：

  1. 对象字面量 优点：简单易用，适用于创建单个对象。

缺点：不方便复用，当需要创建多个类似对象时，需要重复编写。

  1. 构造函数 优点：可复用，可以创建多个类似的对象。

缺点：构造函数的方法和属性不能被复用，存在内存浪费和代码冗余的问题。

  1. 原型 优点：共享方法和属性，避免了重复的内存开销，提高了代码的复用性。

缺点：对于引用类型，容易出现共享数据的问题，需要在实现时特别注意。

  1. ES6 的 class 优点：语法更加简洁易读，符合面向对象编程的思想。

缺点：需要编译成 ES5 代码才能在老版本浏览器中使用，不够灵活。

在实际开发中，通常可以结合使用不同的对象创建方式，以满足不同的需求。例如，可以使用对象字面量创建一些简单的对象，使用构造函数或 class 来创建更加复杂的对象，使用原型来实现共享属性和方法。

2.原型和原型链

JavaScript 只有一种结构：对象。每个实例对象（object）都有一个私有属性（称之为 proto）指向它的构造函数的原型对象（prototype）。该原型对象也有一个自己的原型对象（__proto__），层层向上直到一个对象的原型对象为 null。根据定义，null 没有原型，并作为这个原型链中的最后一个环节。

原型（Prototype）是 JavaScript 中每个对象都具有的一个内部属性，它指向另一个对象，这个对象就是原型对象（Prototype Object）。原型对象又具有原型，通过这种方式，一直追溯到 Object.prototype，这就是原型链（Prototype Chain）。

当访问一个对象的某个属性时，JavaScript 引擎会先查找对象本身是否有这个属性，如果没有，就会去它的原型对象上查找，如果还是没有，就会去原型对象的原型对象上查找，以此类推，直到找到 Object.prototype，如果还没有找到，则返回 undefined。这样，每个对象都会和一个原型对象联系起来，形成一个原型链。

可以通过__proto__属性来访问一个对象的原型，通过Object.getPrototypeOf(obj)方法也可以获取对象的原型。另外，使用构造函数创建对象时，每个对象会自动关联上构造函数的原型对象
